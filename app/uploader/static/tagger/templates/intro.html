Introduction (Click anywhere to hide this)

This page allows you to tag elements of interest in your XML document to use in the analysis. When done mapping the document, click Save and Return to save the document structure file to your project. The results of this the tagging will be saved in a a '##_structure.json' file. The file will not replace any previously created files.

Wordseer elements types:

  • Text nodes are the XML nodes that contain the content text to analyze (e.g. book paragraphs, quotes, stanzas...etc.).
    • Click  Text to add a text node.
    • The are can be any number of text nodes.
    • Text nodes are called sentences by default and cannot be renamed.

  • Property Nodes are XML nodes that contain additional descriptive information (or meta data) about the text (e.g. text type, author, line number...etc.).
    • Click  Property to add a property node.
    • Property nodes can be assigned a display name (i.e. renamed).
    • Some Property nodes can be assigned titles from other nodes (see below).
    • Property nodes can be edited at any time by clicking on a node in the properties bucket ().

XML elements types:

XML Elements, or nodes, are represented in a tree of different types.

  • The XML document root element is added by default and is treated as a property node (i.e. can be assigned a title and renamed).
  • Nodes in light blue are either leaf nodes or attributes of other nodes. These nodes usually contain text and can be either text nodes or properties.
  • Nodes in dark blue are XML tree nodes that have child nodes but no content of their own. Such nodes are usually property nodes and can be assigned title text from a child nodes when it is added as a property.

In certain cases, you might want to choose a node with children as a text node (e.g. if the document contains formatting tags that should be ignored). In this case, all the text will be extracted from the child nodes into the text node content.

Notes:

  • A node can be either a text node or property node, not both.
  • Some XML tag names have a semantic meaning (e.g. Place or Name). You can use this name as an additional property for your analysis.
